# Social Content Plan

Built for turning a topic or launch into a small social content package with platform choices, hooks, and post drafts.

## What this skill should deliver

- A platform-aware content plan tied to one goal and one audience.
- A small set of hooks or post angles that vary by platform.
- Drafts or outlines for multiple posts rather than one isolated message.
- A lightweight distribution pack that includes community or comment strategy plus repurposing assets.

## Preferred approach

1. Clarify the user's goal, inputs, and required deliverable.
2. Read [references/usage-notes.md](references/usage-notes.md) before acting.
3. Produce one concrete output before adding explanation.
4. Use the following operating rules:

- Choose platforms intentionally instead of spreading one generic post everywhere.
- Lead with the hook and angle before the body draft.
- Repurpose one core idea into multiple platform-native forms.
- Keep each post aligned to one primary action or outcome.
- Separate the primary audience from secondary audiences instead of treating everyone as one blob.
- Name the supporting asset or follow-up move that makes the post more executable: demo, carousel, short clip, comment link, or community post.
- If the prompt uses placeholders like `[Feature Name]` or `[Problem]`, replace them with one plausible concrete stand-in instead of leaving the plan abstract.
- When the launch spans multiple channels, give a simple rollout cadence so the team knows what to post first and next.

## Output format

- Start with a `Goal` section that distinguishes primary and secondary outcomes when relevant.
- Then provide an `Audience ladder` section showing who the content is trying to reach first and second.
- Then provide a `Platform plan` table with platform, format, and why it fits.
- Then add a `Rollout cadence` section with the order or day-by-day release sequence.
- Then add a `Community / distribution notes` section for non-primary channels such as comments, communities, or newsletter reuse.
- Then add `Hook options` with several post angles.
- Then provide `Drafts` for 3 platform-specific posts.
- Then add an `Asset list` section describing what supporting artifact each post needs.
- End with a short `Repurposing notes` section.

## Avoid

- Do not write the same post three times with only small wording changes.
- Do not ignore platform differences in tone or format.
- Do not turn the plan into generic social media advice.
- Do not leave the plan without a concrete distribution move beyond the post text itself.
- Do not leave bracket placeholders untouched when a concrete stand-in would make the pack more immediately usable.

## Inputs

- Natural-language user request
- Referenced files or URLs
- Existing project context, if available

## Outputs

- A concrete deliverable, recommendation, or implementation result
- Short notes on assumptions, caveats, or next actions when needed

## Edge Cases

- If required inputs are missing, state exactly what is missing.
- If the request only partially matches this skill, handle the matching portion and clearly scope the rest.
- If a risk, safety, or compliance concern appears, surface it before producing the final output.
