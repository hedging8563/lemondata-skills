# Usage Notes

This reference file supports `writing-postmortem-brief`.

## When this skill is most useful

- A concise executive summary that explains what happened and why it mattered.
- A timeline that makes the incident progression clear.
- A blameless root cause analysis with contributing factors.
- Action items that have owner and priority, not just generic lessons.

## Delivery hints

- A useful postmortem should help the next responder act faster, not just document the past.
- Root cause analysis is stronger when timeline facts stay separate from the interpretation.
- Action items should be concrete enough to become tickets immediately.
