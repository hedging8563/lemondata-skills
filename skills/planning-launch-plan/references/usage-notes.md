# Usage Notes

This reference file supports `planning-launch-plan`.

## When this skill is most useful

- A launch goal with a clear audience and success target.
- A milestone plan that organizes work into a manageable sequence.
- A short risk register with mitigation and ownership.
- Immediate next actions with concrete owners or time anchors.

## Delivery hints

- Milestones should be sequenced for decision-making, not only chronology.
- A launch plan is better when owners and success signals are explicit.
- Risks should include both execution risk and go-to-market risk where relevant.
