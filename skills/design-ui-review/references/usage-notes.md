# Usage Notes

This reference file supports `design-ui-review`.

## When this skill is most useful

- A top-line judgment on what is working and what is hurting clarity or conversion.
- A prioritized issue list with severity and rationale.
- A short design direction that tells the team what to change first.

## Delivery hints

- The best UI review tells the team what to do next, not just what is wrong.
- Severity should reflect user impact, not reviewer preference.
- Quick wins should be changes the team can implement without redesigning the whole screen.
